## ‚öôÔ∏è PARAMETER GUIDE

### Stylize Values (--s)
Controls artistic interpretation vs. prompt adherence

- `--s 200`: Very literal, precise (Technical Blueprint)
- `--s 250`: Balanced, clean execution (All-purpose)
- `--s 275`: Slight artistic interpretation (Fantasy-Tech)
- `--s 300-350`: More artistic, dramatic (Sci-Fi Futuristic)

**Recommendation by Direction:**
- **Technical Blueprint:** --s 200-250 (precision critical)
- **Sci-Fi Futuristic:** --s 300-350 (drama encouraged)
- **Grounded Fantasy-Tech:** --s 275 (balanced)

### Version Flags (--v)
- `--v 7`: Midjourney V6 (stable, widely documented for UI/branding)
- `--v 7`: Midjourney V7 (newest, improved quality, supports personalization)

**Recommendation:** Use `--v 7` with `--p` for personalized style consistency across all generations

### Style Flags
- `--style raw`: More literal interpretation, less artistic (use for Technical Blueprint)
- No flag: Standard artistic interpretation (Sci-Fi, Fantasy-Tech)

## üîç TROUBLESHOOTING

### Common Issues & Fixes

#### Issue: UI elements look too photorealistic
**Fix:** Add "flat design, vector illustration, UI mockup style, no photography"

#### Issue: Text in mockups is illegible
**Fix:** Add "clean typography visible, readable text, crisp letterforms" or generate without text and add in Figma

#### Issue: Colors don't match brand palette exactly
**Fix:** Midjourney approximates colors. Use hex codes in prompt but expect slight variance. Adjust in post-production.

#### Issue: Too many random elements
**Fix:** Add "minimal composition, clean layout, organized structure"

#### Issue: Perspective is wrong
**Fix:** Specify exactly: "isometric 30-degree angle" or "front elevation view" or "top-down view"

#### Issue: Looks too generic/stock
**Fix:** Add unique style references: "distinctive design language, bold unique aesthetic, memorable visual identity"
